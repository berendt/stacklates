Introduction to stacklates.com

Heat, the OpenStack orchestration service, provides a template based orchestration for describing cloud applications by executing appropriate OpenStack API calls to generate running cloud applications.

There is a official repository available at openstack/heat-templates providing a handful of sample Heat Orchestration Templates (HOT). To update or add templates to the repository it is necessary to use the Gerrit workflow, which is good but rather complex, especially for newcomers.

stacklates.com tries to provide a comfortable and easy to use platform to share and work with templates for Heat.

heat_template_version: 2013-05-23

description:
  # a description of the template

parameter_groups:
  # a declaration of input parameter groups and order

parameters:
  # declaration of input parameters

resources:
  # declaration of template resources

outputs:
  # declaration of output parameters
